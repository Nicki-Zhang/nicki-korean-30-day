# Nikigo Course Engine Adaptation Matrix

## Review basis

- Source review: `lesson-00`, `lesson-04`, `lesson-06`, `lesson-07`, `lesson-10`, `lesson-11`, their engines, `course-catalog.js`, `app-state.js`, and `audio-catalog.js`.
- Browser review: the real pages were opened at 390 CSS px through Playwright. Runtime IDs, headings, progress counts, enabled and disabled actions, gate notices, and horizontal overflow were checked.
- This document describes the existing behavior. It does not propose changes to answers, content, XP, review scheduling, audio resolution, or completion gates.

## Shared findings

1. The existing engines already own all answer, mistake, completion, XP, audio, and session behavior. A Shell adapter should only read their view state and provide slots.
2. The common session convention is `nikigoLessonSession:<stableId>`. Lesson 0 is intentionally outside it.
3. Course-local retry is not the same as the global `app-state.js` ReviewItem schedule. The Shell must not merge those concepts.
4. The seven-part teaching model can be represented as optional phase metadata. Current lessons contain 13, 15, or 18 real steps, while Lesson 0 has no linear steps.
5. Audio state must remain a semantic input from the existing resolver or gate. The Shell may display `available`, `pending`, or `blocked`, but may not calculate those states itself.
